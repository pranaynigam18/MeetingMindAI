# Engineering Rules

## Architecture

Keep the architecture simple.

Avoid unnecessary folders.

Avoid unnecessary files.

Reuse existing services whenever possible.

Never rewrite working code.

Modify only the required files.

---

## Python

Use FastAPI.

Write readable Python.

No unnecessary abstractions.

No duplicated code.

Always include imports.

---

## Database

Use PostgreSQL.

Never duplicate data.

Always use parameterized SQL.

---

## UI

Professional.

Minimal.

Inspired by:

- Notion
- Linear
- Stripe

No flashy design.

White background.

Dark navigation.

Clean cards.

---

## AI

Reuse existing Gemini service.

Reuse Whisper.

Don't replace working services.

---

## Code Quality

Every response must contain:

1. Files modified

2. Why they were modified

3. Complete code

Never return partial code.
